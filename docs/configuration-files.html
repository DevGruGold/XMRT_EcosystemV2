4. Project README

File: /xmrt-ecosystem-v2/README.md
# XMRT Ecosystem V2

## Overview

The XMRT Ecosystem V2 is a comprehensive cross-chain multi-resource token platform that enables seamless asset management, AI-powered automation, and decentralized applications across multiple blockchain networks.

## Architecture

```
xmrt-ecosystem-v2/
├── apps/
│   ├── web/           # Next.js web application
│   ├── mobile/        # React Native mobile app
│   └── api/           # Node.js API backend
├── packages/
│   ├── shared/        # Shared utilities and components
│   ├── types/         # TypeScript type definitions
│   └── ui/            # Shared UI components
├── contracts/
│   ├── core/          # Core XMRT smart contracts
│   ├── defi/          # DeFi protocol contracts
│   └── governance/    # DAO governance contracts
├── ai-agents/
│   ├── trading/       # AI trading agents
│   ├── analytics/     # Data analytics agents
│   └── automation/    # Workflow automation agents
├── docs/              # Documentation
└── infrastructure/    # DevOps and deployment configs
```

## Features

- **Multi-chain Support**: Ethereum, Polygon, Arbitrum, Optimism
- **AI-Powered Trading**: Automated trading strategies and portfolio management
- **DeFi Integration**: Yield farming, liquidity provision, and staking
- **Cross-chain Bridges**: Seamless asset transfers between chains
- **Mobile First**: Native mobile applications for iOS and Android
- **Real-time Analytics**: Advanced portfolio tracking and performance metrics

## Quick Start

### Prerequisites

- Node.js 18+
- Docker & Docker Compose
- Git

### Installation

1. Clone the repository:
```bash
git clone https://github.com/xmrt-ecosystem/xmrt-ecosystem-v2.git
cd xmrt-ecosystem-v2
```

2. Install dependencies:
```bash
npm install
```

3. Set up environment variables:
```bash
cp .env.example .env
# Edit .env with your configuration
```

4. Start the development environment:
```bash
npm run docker:up
npm run dev
```

### Development Commands

- `npm run dev` - Start all development servers
- `npm run build` - Build all packages
- `npm run test` - Run all tests
- `npm run lint` - Lint all code
- `npm run format` - Format code with Prettier
- `npm run typecheck` - Check TypeScript types

5. Docker Compose Configuration

File: /xmrt-ecosystem-v2/docker-compose.yml
version: '3.8'

services:
  # PostgreSQL Database
  postgres:
    image: postgres:15
    environment:
      POSTGRES_DB: xmrt_db
      POSTGRES_USER: xmrt_user
      POSTGRES_PASSWORD: xmrt_password
    ports:
      - "5432:5432"
    volumes:
      - postgres_data:/var/lib/postgresql/data
    networks:
      - xmrt_network

  # Redis Cache
  redis:
    image: redis:7-alpine
    ports:
      - "6379:6379"
    volumes:
      - redis_data:/data
    networks:
      - xmrt_network

  # MongoDB for AI Agents
  mongodb:
    image: mongo:7
    environment:
      MONGO_INITDB_ROOT_USERNAME: xmrt_admin
      MONGO_INITDB_ROOT_PASSWORD: xmrt_mongo_password
    ports:
      - "27017:27017"
    volumes:
      - mongodb_data:/data/db
    networks:
      - xmrt_network

  # IPFS Node
  ipfs:
    image: ipfs/go-ipfs:latest
    ports:
      - "4001:4001"
      - "5001:5001"
      - "8080:8080"
    volumes:
      - ipfs_data:/data/ipfs
    networks:
      - xmrt_network

  # Ethereum Local Node (Hardhat)
  hardhat:
    build:
      context: ./contracts
      dockerfile: Dockerfile
    ports:
      - "8545:8545"
    volumes:
      - ./contracts:/app
    networks:
      - xmrt_network

  # API Backend
  api:
    build:
      context: ./apps/api
      dockerfile: Dockerfile
    ports:
      - "3001:3001"
    environment:
      - DATABASE_URL=postgresql://xmrt_user:xmrt_password@postgres:5432/xmrt_db
      - REDIS_URL=redis://redis:6379
      - MONGODB_URL=mongodb://xmrt_admin:xmrt_mongo_password@mongodb:27017
    depends_on:
      - postgres
      - redis
      - mongodb
    networks:
      - xmrt_network

  # Web Frontend
  web:
    build:
      context: ./apps/web
      dockerfile: Dockerfile
    ports:
      - "3000:3000"
    environment:
      - NEXT_PUBLIC_API_URL=http://api:3001
      - NEXT_PUBLIC_CHAIN_ID=31337
    depends_on:
      - api
    networks:
      - xmrt_network

  # AI Agents Service
  ai-agents:
    build:
      context: ./ai-agents
      dockerfile: Dockerfile
    ports:
      - "8000:8000"
    environment:
      - MONGODB_URL=mongodb://xmrt_admin:xmrt_mongo_password@mongodb:27017
      - REDIS_URL=redis://redis:6379
    depends_on:
      - mongodb
      - redis
    networks:
      - xmrt_network

volumes:
  postgres_data:
  redis_data:
  mongodb_data:
  ipfs_data:

networks:
  xmrt_network:
    driver: bridge
